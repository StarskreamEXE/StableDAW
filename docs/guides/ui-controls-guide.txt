UI CONTROLS GUIDE — PLAIN ENGLISH REFERENCE
=============================================

A complete breakdown of every control in the generation interface.
No jargon, no math — just what each knob does, what to try, and what to avoid.

PROMPTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Prompt
  Describe the audio you want. Be specific: genre, instruments, tempo, mood,
  structure. Example: "120 BPM house loop, deep sub bass, crispy hi-hats,
  vinyl crackle"

Negative prompt
  Describe what you DON'T want. Steers the model away from unwanted qualities.
  Example: "poor quality, distortion, vocals, silence"

TIPS:
  • Include tempo (BPM), instruments, genre, mood, and texture
  • The more specific you are, the more the model follows your vision
  • Negative prompt only matters when CFG scale > 1

MAIN CONTROLS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Seconds total [0–512, default: 120]
  How long the generated audio will be. Longer = more VRAM used during
  generation. The model handles variable lengths natively — it doesn't waste
  compute on unused silence.

Steps [1–500, default: 8 for ARC]
  How many refinement passes the model makes. ARC models are built for 8
  steps — going higher won't help much. RF models need ~50. Going below 4 on
  ARC or below 20 on RF sounds noticeably worse.

CFG scale [0–25, default: 1.0 for ARC, 7.0 for RF]
  How strictly the model follows your prompt vs. freestyling.
    1.0 = model does its own thing (ARC default, guidance not needed)
    7.0 = strong prompt adherence (RF default)
   25.0 = over-saturated, likely artifacts

SUGGESTIONS:
  • ARC models: leave Steps at 8, CFG at 1.0 — they're designed for this
  • RF models: start with Steps 50, CFG 7.0 and adjust from there
  • Want more creative freedom? Lower CFG. Want tighter prompt matching? Raise it.

SAMPLER PARAMS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

── Core Sampler Settings ──

Seed [-1 or any integer, default: -1]
  The random starting point. Same seed + same settings = same result every
  time. -1 means random — every generation is unique. Save a seed when you
  love a result and want to recreate or tweak it.

Sampler type [pingpong / euler / rk4 / dpmpp, default: pingpong for ARC]
  The algorithm used to progressively build audio from noise.

    Pingpong  — Fast, great at 8 steps. Default for ARC.
    Euler     — Fastest. Good for quick drafts, RF models.
    DPM++     — Medium speed, better quality per step. Good for RF.
    RK4       — Slowest (4x Euler), most precise. Maximum accuracy.

Sigma max [0–1, default: 1.0]
  Starting noise level. At 1.0 = generation from pure noise. Below 1.0
  (with init audio) = controls how much original audio survives.

Duration padding [0–30 sec, default: 6.0]
  Extra seconds added internally so reverb tails and decays don't get cut
  off. The padding is silent in your output — it just gives the model room
  to breathe at the end.


── Guidance Fine-Tuning ──

APG scale [0–1, default: 1.0]
  Anti-saturation filter for CFG.
    1.0 = full APG: removes harsh/overblown parts of guidance, keeps detail
    0.0 = vanilla CFG with no protection
  Think of it as a "tasteful guidance" knob.

CFG interval min [0–1, default: 0.0]
CFG interval max [0–1, default: 1.0]
  Restricts guidance to only apply during certain stages of generation.
  Default (0–1) = guidance at all stages. Try 0.2–0.8 to skip guidance at
  the noisiest and cleanest stages — can reduce artifacts while keeping
  prompt adherence.

CFG rescale [0–1, default: 0.0]
  Fixes the volume/saturation boost that high CFG causes.
    0.0 = no correction
    1.0 = full correction
  Turn this up if high CFG makes your outputs overblown or clipping.

CFG norm threshold [0–100, default: 0.0]
  Safety valve that caps how large the guidance correction can get.
    0.0 = disabled
    Higher values prevent sudden spikes that cause glitches.
  Use if you hear random pops or distortion with high CFG.

SUGGESTIONS:
  • For ARC at CFG 1.0: these don't matter — guidance isn't active
  • For RF at CFG 7+: start with APG 1.0, if too aggressive try 0.5
  • If you hear artifacts with high CFG: raise CFG rescale to 0.3–0.7


── Sampling Schedule Shift ──

Controls how the model distributes its effort across the generation process.

Schedule shift type [LogSNR / Flux / Full / None, default: LogSNR]
  Changes where the model focuses its denoising work.

    None    — Equal effort at all stages
    LogSNR  — More focus on structure-forming stages (log curve)
    Flux    — More effort at high-noise stages where structure forms
    Full    — Applies both training and sampling shifts together

Sub-parameters (only visible for the selected type):

  LogSNR:
    Anchor length [100–10000]     — Reference sequence length for the curve
    Anchor log-SNR [-12 to 0]     — Effort on structure-forming stages
    Rate [-2 to 2]                — How quickly shift scales with duration
    log-SNR end [-2 to 6]         — Where the shift tapers off

  Flux:
    Min seq len [1–10000]         — Shortest calibrated sequence
    Max seq len [1–10000]         — Longest calibrated sequence
    Alpha min [0.1–20]            — Shift strength at short durations
    Alpha max [0.1–20]            — Shift strength at long durations

  Full:
    Base shift [0–5]              — Minimum shift at short sequences
    Max shift [0–5]               — Maximum shift at long sequences
    Min length [1–10000]          — Where base_shift applies
    Max length [1–10000]          — Where max_shift applies

SUGGESTION: Leave at defaults unless output sounds mushy (try more shift) or
over-processed (try less). These are advanced — defaults are tuned.

INIT AUDIO (AUDIO-TO-AUDIO)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Upload an existing audio file and use it as a starting point. The model will
transform it guided by your prompt.

Init audio [upload]
  The source recording to transform.

Init noise level [0.01–1.0, default: 0.9]
  How much of the original survives.

    0.1 = Subtle remix — like adding effects to a finished track
    0.3 = Loose cover — same structure, different performance
    0.5 = Inspired by — recognizable DNA but a new piece
    0.9 = Sampled one note, built something new
    1.0 = Original ignored entirely

SUGGESTIONS:
  • Start at 0.7–0.9 for creative variations
  • Use 0.1–0.3 to preserve rhythm/structure while changing texture
  • Your prompt steers WHAT the transformation becomes


── RF-Inversion (RF models only) ──

A more precise version of audio-to-audio. Instead of simply adding noise to
your audio, the model first "reverse-engineers" it back into noise space more
carefully, then re-generates. Preserves fine timing and transient details.

Inversion Steps [1–500, default: 100]
  How carefully the model "listens" to your input.
  More = more faithful to source structure.

Gamma [0–1, default: 0]
  Creative freedom during inversion.
    0   = maximum faithfulness
    0.3 = slight liberty
    1.0 = maximum reinterpretation

Unconditional [On/Off, default: Off]
  When ON: inversion ignores text prompt, reconstructing purely from audio
  structure. Prompt only affects re-generation. Gives cleaner separation
  between "what to keep" and "what to change."

SUGGESTIONS:
  • For reproduction: empty prompt, CFG 1, gamma 0.3
  • For prompt re-stylization: CFG 1–7, gamma 0–0.15, unconditional checked

INPAINTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Regenerate only a specific section of an audio file while keeping everything
else untouched. Like highlighting a section in your DAW and saying "redo just
this part."

Inpaint audio [upload]
  The source audio to partially regenerate.

Mask Start (sec) [0–duration, default: 0]
  Where the region to regenerate begins. Everything before is preserved.

Mask End (sec) [0–duration, default: duration]
  Where the region to regenerate ends. Everything after is preserved.

USE CASES:
  • Fix a flubbed section or awkward transition
  • Replace a weak chorus while keeping verse and bridge
  • Swap an instrument in a specific bar

CONTINUATION (extending audio):
  Set Mask Start to the end of your existing audio and Mask End beyond it
  (with a longer Seconds Total). The model generates new material that flows
  naturally from where yours left off — like asking a session musician to
  improvise an outro matching your intro.

LORA (STYLE PATCHES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Small add-on "style patches" trained on specific audio (your own samples, a
genre, a mood) that steer the model's output without retraining the whole
thing. Only visible when LoRAs are loaded.

Strength [0–10, default: 1.0]
  How much the style patch influences output.
    0   = off
    1.0 = full effect
    >1  = amplified/exaggerated

Interval min [0–1, default: 0.0]
Interval max [0–1, default: 1.0]
  At which generation stages the LoRA is active.
    Early stages (near 1.0) = broad style/structure
    Late stages (near 0.0)  = fine texture/detail

Layer filter [text, default: empty]
  Which internal model layers the patch affects. Leave empty for all layers.
  Advanced use only.

SUGGESTIONS:
  • Start with strength 1.0, full interval (0–1)
  • If the LoRA is too dominant, reduce strength to 0.3–0.7
  • To apply only to early structural stages: set interval to 0.5–1.0
  • Multiple LoRAs can be stacked and controlled independently

OUTPUT PARAMS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

File format [wav/flac/mp3 320k/mp3 v0/mp3 128k/m4a 64k/m4a 32k, default: wav]
  Output audio format.

    WAV       — Perfect quality (lossless), large files. Final masters.
    FLAC      — Perfect quality, ~50% smaller than WAV. Archiving.
    MP3 320k  — Near-perfect, small. Sharing, nearly indistinguishable from WAV.
    MP3 V0    — Near-perfect (variable bitrate), slightly smaller than 320k.
    MP3 128k  — Noticeable quality loss. Rough demos only.
    M4A 64k   — Acceptable for speech. Podcasts, streaming.
    M4A 32k   — Minimum viable. Extreme bandwidth saving only.

File naming [verbose/prompt/output.wav, default: verbose]
    Verbose   — Prompt + settings + seed in filename. Best for experiments.
    Prompt    — Just the prompt text. Cleaner but less info.
    output.wav— Generic name. Quick single generations.

Cut to seconds total [On/Off, default: On]
  Trims output to exactly your requested duration, removing padding silence.

Autoplay [On/Off, default: Off]
  Plays audio automatically when generation finishes.

Infinite Radio [On/Off, default: Off]
  Continuously generates new clips back-to-back — endless AI music stream.

Auto Download [On/Off, default: Off]
  Saves each generation to your downloads folder automatically.

Spec Preview Every [0–100, default: 0 (disabled)]
  Shows a visual spectrogram every N steps so you can watch the audio take
  shape during generation. A spectrogram is a picture of sound showing
  frequencies over time.

OUTPUT SECTION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Output audio
  Plays back the generated audio.

Output spectrogram
  Visual representation — shows frequencies over time, useful for spotting
  issues at a glance.

Send to init audio [button]
  Takes your generated output and loads it as init audio for audio-to-audio
  workflows.

Send to inpaint audio [button]
  Takes your generated output and loads it for inpainting/continuation.

WORKFLOW TIP:
  Generate something you like -> Send to init audio -> adjust noise level
  and prompt -> generate a variation. Iterate until perfect.
